# Implementation Notes

## Important constraints
- Do not start with complex integrations.
- Do not overfit to a single monolithic template.
- Do not couple email and PDF rendering to the hosted web renderer.
- Do not make vector search a dependency for core publishing.

## Important guidance
- Use the provided HTML as a section inventory and field reference.
- Use the sample newsletter files as visual inspiration for polish and editorial hierarchy.
- Keep rendering configuration-driven.
- Keep the content model versioned.
- Prioritize speed to first useful draft.
