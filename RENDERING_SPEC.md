# Rendering Spec

## Core rule
Never treat one HTML layout as the master output for every channel.

## Web renderer
- responsive
- interactive
- school-branded
- SEO-friendly
- hosted and shareable

## Email renderer
- simplified
- linearized where necessary
- email-safe structure and CSS
- hosted image URLs
- include subject and preview text fields in distribution UI

## PDF renderer
- page-aware
- stable layout
- stronger editorial look
- clean page breaks
- print-safe assets

## HTML export renderer
- standalone HTML
- reusable in external websites or CMS
- no dependency on app-specific chrome

## Blog/website renderer
- article-friendly
- clean structure for publishing or copy/export
